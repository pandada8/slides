看起来分工很美好

Problems

  • 代码中存在大量约定的内容
  • 前端工作（JS/CSS）依赖于后端产生的 HTML
  • 数据来源混杂：一部分数据在 DOM 上，一部分数据要 ajax 获取
  • 业务逻辑混乱：一些在前端，一些在 Model 层，更多的在 View层

问题不只有这些

  • 前端需要后端的开发环境
  • 服务端 View 层高度耦合
  • 沟通成本高
  • 职责不清晰

谁来维护页面？

前端写 Demo，后端套页面
  • 后端需要写 HTML
  • 前端要确认后端写的 HTML 没问题
前端写页面，后端负责数据
  • 前端需要了解后端语言和后端架构

有什么好处呢?

  • 关注点分离
  • 职责分离
  • 更好的共建模式
  • 快速的反应变化

new Vue({
	el: "#demo2",
	data: {
		tasks: [],
		new_name: ""
	},
	methods: {
		delete: function(index){
			this.tasks.splice(index, 1);
		},
		add: function(){
			if(this.new_name.length){
				this.tasks.push(this.new_name)
				this.new_name = ""
			}
		}
	}
})
